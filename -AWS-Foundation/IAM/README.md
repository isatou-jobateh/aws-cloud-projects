# AWS IAM Lab

## Overview

In this lab, I explored the fundamentals of AWS Identity and Access Management (IAM) using AWS Skill Builder. I learned how IAM controls access to AWS resources by using users, groups, policies, and roles.

## Topics Covered

- Explored pre-created IAM users and groups
- Inspected IAM policies applied to user groups
- Added users to groups based on job responsibilities
- Located and used the IAM user sign-in URL
- Tested how IAM policies affect access to AWS services

## Skills Gained

- AWS Identity and Access Management (IAM)
- IAM Users
- IAM Groups
- IAM Policies
- IAM Roles
- Principle of Least Privilege
- Access Control
- Permission Management

## What I Learned

- IAM allows secure management of access to AWS resources.
- Policies define what actions users, groups, and roles are allowed or denied.
- Assigning permissions to groups is easier than assigning permissions to individual users.
- IAM Roles provide temporary credentials without sharing long-term access keys.
- Following the Principle of Least Privilege improves security by granting only the permissions required.
